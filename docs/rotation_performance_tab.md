# SG Rotation Performance Tab

This document describes the `Performance` tab in the frontend and the report
that powers it.

## Purpose

The tab gives a more detailed view of SG Rotation Leipzig II match performance
than the general league table or promotion-race analysis.

It is intended to answer questions like:

- When do we score?
- When do we concede?
- How strong are we after scoring first?
- How badly do we suffer after conceding first?
- Do we protect halftime leads?
- Are late goals for or against a pattern?
- In which match states are we stable, and where are we fragile?

## Report source

The tab loads:

- `reports/rotation_match_performance.json`

This file is generated by:

- `scripts/analyze_rotation_match_performance.py`

The generator is called from:

- `scripts/build_app_data.sh`

The frontend copy step is:

- `scripts/copy_data.sh`

## Data rules

The report intentionally separates two data layers:

1. Canonical result layer
   - Source: `output/app_data/fixtures.json`
   - Used for overall record, points, home/away splits, clean sheets, 2+ goals,
     2+ conceded, and other result-driven metrics.

2. Trusted event layer
   - Source: `output/match_details/*.json`
   - Used for goal timing, first-goal splits, halftime-state derivation,
     lead-loss detection, comeback detection, cards, substitutions, attendance,
     and match drilldown timelines.
   - A match-detail file is only used if its parsed goal events exactly match
     the final score, or if it is a valid `0:0` with no goal events.

This prevents the frontend from using incorrect fallback scores from detail
pages that do not have a reliable event timeline.

## What the tab shows

### 1. Snapshot

High-level KPIs:

- overall record
- points and PPG
- goals for / against
- clean sheets
- failed-to-score count

### 2. Headline findings

Short derived insights from the current dataset, for example:

- record after scoring first
- record after conceding first
- late-goal weakness
- halftime control

### 3. Goal timing

Minute-bin distributions for:

- goals scored
- goals conceded

Bins:

- `0-15`
- `16-30`
- `31-45`
- `46-60`
- `61-75`
- `76-90`

Also shown:

- average minute of first goal scored
- average minute of first goal conceded
- second-half goal share
- late-goal share

### 4. Game-state reaction

Split records for:

- scored first
- conceded first
- level/no-goal timeline
- ahead at halftime
- level at halftime
- behind at halftime

Additional stability / reaction counters:

- dropped points after leading
- points won after trailing
- average response time to equalize

### 5. Stability metrics

Records when:

- keeping a clean sheet
- scoring 2+ goals
- conceding 2+ goals

This is the main section for identifying stable vs fragile match profiles.

### 6. Home / away splits

Separate summary blocks for:

- home matches
- away matches

Each includes:

- record
- PPG
- GF/game
- GA/game

### 7. Match explorer

Each played match gets a drilldown card with:

- date and opponent
- home/away marker
- result
- halftime score
- first-goal side
- goal timeline
- cards
- substitutions
- attendance
- venue
- derived flags like clean sheet, failed to score, lead dropped, comeback to points

From the repo root:

```bash
bash scripts/build_app_data.sh
bash scripts/copy_data.sh
```

Then, if needed:

```bash
cd frontend
npm run ci
```

## Limitations

- Player names remain obfuscated by fussball.de and are not used.
- Event-based metrics depend on trusted match-detail coverage.
- `0:0` matches count for result metrics but not for first-goal or minute-of-goal
  charts.
- The current implementation is Rotation-specific, not a generic all-team module.
